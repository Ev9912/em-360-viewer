EM 360 Image Viewer

User guide — Phase 1

Requirements

Browser: Google Chrome or Microsoft Edge (Chromium-based). Firefox and Safari are not supported.

Images: Equirectangular JPG or PNG panoramas. Standard output from Ricoh Theta, Insta360, GoPro Max, and similar cameras.

Running the app: Open start-server.bat in the app folder first, then navigate to localhost:8000 in Chrome or Edge. ES modules require a local server — the app will not work if opened directly as a file.

Opening a Folder

  1. Click Open Folder on the welcome screen.
  2. Select the folder containing your 360° images. Subfolders are automatically included — each subfolder becomes a separate group.
  3. Chrome will show a permission prompt: "Let site edit files?" — click Edit files. This is required so the app can save your last-viewed position between sessions.
  4. The app scans the folder, reads EXIF data, and opens the first image. On subsequent opens of the same folder this is nearly instant (data is cached).

Last-viewed frame: When you reopen a folder, the app automatically returns to the last image you were viewing.

Navigation

Moving between images

← → arrow keys Previous / next image ← → buttons Previous / next image (on-screen) Click a thumbnail Jump directly to any image

Looking around inside an image

Click & drag Pan the view in any direction Scroll wheel Zoom in / out Pinch (touch) Zoom in / out on touch screens

View state is preserved. When you move to a different image and come back, your pan and zoom position is restored exactly as you left it.

Control Bar

HDG — Heading Lock

When HDG is active (highlighted blue), each new image opens pointing in the compass direction recorded by the camera at capture time (from EXIF GPano data). Use this when walking a straight corridor or comparing images shot in the same direction.

When HDG is off, your current pan angle carries forward to the next image — useful when you want to keep looking at the same wall or feature as you step through a sequence.

Compass rose

The compass in the top-right corner of the viewer rotates as you pan. North is the direction the camera recorded as north at capture time.

Frame counter and filename

The counter in the bottom-left shows your current position in the folder (e.g., 14 / 47). The filename is shown in the centre of the control bar.

Change Folder

Opens a new folder picker without reloading the page. Your view state from the previous folder is discarded.

Thumbnail Strip

The strip along the bottom of the viewer shows all images in the current folder. Thumbnails load lazily as you scroll — the strip handles large folders without slowing down the initial load.

The active image is highlighted with a blue border. Clicking any thumbnail jumps to that image immediately.

Sort Order

Images are sorted by capture time (EXIF DateTimeOriginal) when available. If capture time is absent or identical, images fall back to a natural filename sort (IMG_001, IMG_002 … IMG_010 rather than alphabetical).

Cache File (.tour.json)

The first time you open a folder, the app writes a hidden file called .tour.json into that folder. It stores the image list, EXIF data, and your last-viewed position.

On subsequent opens, this file is used instead of re-reading EXIF from every image — making reopens near-instant for large folders. If images are added or removed, the cache is automatically discarded and a fresh scan runs.

SharePoint / OneDrive: If the folder is synced to the cloud, .tour.json will sync alongside the images. It is a small text file (~5 KB for 100 images) and safe to leave in place. It does not modify the images themselves.

Installing as a Desktop App (PWA)

With the app open in Chrome or Edge, look for the install icon (⊕) in the address bar and click it. This adds EM 360 Image Viewer to your Start Menu and taskbar. It opens in its own window with no browser UI — like a native app.

The installed version still requires start-server.bat to be running in the background. A quick way to handle this is to add a shortcut to start-server.bat to your Startup folder so it runs automatically at login.

To add to Windows Startup: press Win + R, type shell:startup, and paste a shortcut to start-server.bat into the folder that opens.

Sharing & Installing on Other Computers

Static web hosting Recommended

Upload the app folder to a static host (Netlify, GitHub Pages, or an internal IIS/nginx server). Colleagues open the URL in Chrome and click the install button — no files to distribute, no setup required. The app caches itself after the first load and works offline.

ZIP & local launcher Offline / air-gapped

Zip the app folder and send it. The recipient unzips it, double-clicks start-server.bat (requires Python 3 or Node.js), then opens localhost:8000 in Chrome. From there they can also install it as a PWA. Python 3 is included with most Windows 11 installs via the Store.

Windows installer (.exe) Phase 2 — not yet built

A future Tauri build will produce a proper Windows installer with a Start Menu entry, no server required, and optional "Open With" file association for JPG files.

SharePoint document library: Uploading the files to a SharePoint document library and opening them directly will not work — SharePoint does not serve JavaScript modules with the correct MIME type. Use a dedicated static host or the ZIP method instead.

Tips

  • Large folders: The first open of a folder with 200+ images takes 10–30 seconds while EXIF is read. Subsequent opens are instant (cache hit).
  • Subfolders: Each immediate subfolder becomes its own group. Deeply nested folders (more than one level down) are ignored.
  • Non-360 images: JPG and PNG files that are not equirectangular panoramas will still open — they'll just look distorted in the spherical viewer. There is no automatic filtering by aspect ratio.
  • HEIC / iPhone: HEIC format is not supported in Phase 1. Convert to JPG before use.
  • Multiple monitors: The installed PWA window can be moved and resized freely. PSV renders at full resolution regardless of window size.
